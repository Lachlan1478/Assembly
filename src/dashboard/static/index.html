Assembly takes a problem description and runs it through a multi-phase structured debate between AI personas, each with different viewpoints and expertise. The system extracts ideas as they emerge, tracks belief evolution, and optionally runs a convergence phase to produce a polished commercial spec.

The mode sets default turns and persona count. fast uses full_history memory (all exchanges verbatim, no extra LLM calls). medium/standard/deep use structured memory — a maintained summary that reduces context bloat as conversations grow long.

Phases are not hardcoded — the LLM generates domain-specific phases tailored to the inspiration. Each phase includes: phase_id, goal, desired_outcome, max_turns, and phase_type. Phase selection strategies (bookends, bookends_plus_middle, all) determine which phases to run.

PersonaManager generates fresh personas per phase, tailored to the domain and phase goal. Personas are cached for reuse across sessions. Each has a name, archetype, purpose, and intellectual strengths that shape how they engage.

FacilitatorAgent decides who speaks next based on novelty scores and recent participation. A gap nudge fires every 4 turns to push unexplored angles. max_turns per phase is enforced. All async tasks (extraction, memory) run in parallel via asyncio.gather().

full_history mode dumps the entire conversation each turn — simple but grows large. structured mode maintains these 3 components updated each turn via update_shared_memory_async(), update_summary_async(), and update_belief_state_async(). This reduces context bloat while preserving continuity.

The mediator is a neutral persona with access to all belief states. It fires when the FacilitatorAgent detects stagnation, circularity, or implicit convergence. Each mediation is logged with sub-types: questions_asked, circular_arguments_detected, synthesis_points, scenarios_presented.

Idea extraction runs asynchronously in parallel with the conversation. Each extracted idea has a status (in_play → rejected or integrated). Personas update certainty, concessions, and deltas each turn — visible in the Personas tab.

The optional convergence phase runs a final LLM synthesis over all raw ideas and conversation logs. It produces a commercial-grade structured spec: product name, one-sentence pitch, target ICP, monetisation model, key differentiator, MVP features, risks, and a 7-day execution plan.
